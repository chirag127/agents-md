# Cline rules — Chirag Singhal

> **Read [`~/AGENTS.md`](file://~/AGENTS.md) first** — that file holds the shared rules across every coding agent. This file adds Cline-specific rules on top: model defaults, known bugs and workarounds, edit-mode preferences, and the skills / MCP servers installed for this agent.

## Model defaults — which models to use

- **Primary:** `claude-sonnet-4-5` via Anthropic API key. Best balance of cost and tool-use reliability for Cline's diff-edit flow.
- **Reasoning / hard tasks:** `claude-opus-4-5` only when Sonnet repeatedly fails on the same step (architecture sketches, tricky refactors, gnarly types). Watch the cost meter — Cline streams the full transcript per turn.
- **Cheap fallback:** `gemini-2.5-pro` via free Gemini API key from `chirag127/secrets` for routine edits when Anthropic quota is tight.
- **Local / offline:** Ollama with `qwen2.5-coder:14b` configured but rarely used; quality drops sharply on multi-file edits.
- **Avoid for Cline:** non-tool-trained models (base Llama, older Mistral) — Cline's `replace_in_file` tool needs strict JSON tool-call adherence or it loops.

## Known bugs / quirks / workarounds

- **ASR-blocked native postinstalls:** Windows Defender Exploit Guard (ASR rule "Block executable files from running unless they meet a prevalence, age, or trusted list criterion") kills unsigned `.exe` postinstalls from npm. Symptom in Cline: `execute_command` shows the install "succeeded" but the binary is gone. Workaround: run installs in a Git Bash terminal outside Cline, or temporarily exclude the project dir in Defender, then re-run.
- **`replace_in_file` whitespace drift:** Cline's diff tool fails silently when the SEARCH block has tab/space mismatches with the file. Symptom: "no match found" loop. Workaround: ask it to fall back to `write_to_file` for that one file, or paste the exact bytes into the search block.
- **Plan→Act handoff loses context on long sessions:** after ~50 turns, switching from Plan to Act sometimes drops the plan. Workaround: paste the agreed plan into the first Act message verbatim.
- **MCP server auto-approve resets:** the auto-approve toggles per MCP tool reset on VS Code reload. Re-enable after every restart for trusted read-only tools (filesystem read, github read).
- **Terminal output truncation:** long `pnpm install` / `uv sync` outputs get cut. Pipe to a file (`> /tmp/install.log 2>&1`) and have Cline read the tail.

## Edit-mode and tool preferences

- **Plan mode first, always.** Per `~/AGENTS.md` "propose, wait, then act" — start every non-trivial task in Plan mode. Switch to Act only after I confirm the plan.
- **Auto-approve allowlist (safe):** read file, list files, list code definitions, search files, MCP read-only calls. Everything else requires explicit approval per action.
- **Never auto-approve:** `execute_command`, `write_to_file`, `replace_in_file`, `browser_action`, any MCP write tool, any network-egress tool.
- **Diff edits over full writes.** Prefer `replace_in_file` over `write_to_file` to keep diffs reviewable; only fall back to full writes when the diff tool fails twice on the same file.
- **Checkpoints on:** Cline's checkpoint feature stays enabled — it's the cheapest undo when a multi-file edit goes sideways.
- **Context window:** keep `.clinerules` short; the file is injected into every turn.

- **OKF knowledge bundles:** per `~/AGENTS.md`, capture durable repo knowledge (schemas, runbooks, metrics, decisions) as `knowledge/` OKF v0.1 bundles. In Cline, load concept files via `read_file` as passive context before tackling a task.
- **MCP servers configured** (in `cline_mcp_settings.json`):
  - `filesystem` — `npx -y @modelcontextprotocol/server-filesystem C:\D` — scoped to the dev root, read+write.
  - `github` — `npx -y @modelcontextprotocol/server-github` — token from `chirag127/secrets`, read-only auto-approved.
  - `context7` — `npx -y @upstash/context7-mcp` — library docs lookup, free tier.
