# Cursor IDE rules — Chirag Singhal

> **Read [`~/AGENTS.md`](file://~/AGENTS.md) first** — that file holds the shared rules across every coding agent. This file adds Cursor-specific rules on top: model defaults, known bugs and workarounds, edit-mode preferences, and the skills / MCP servers installed for this agent.

## Model defaults — which models to use

- **Default for Composer / Agent mode:** `claude-sonnet-4.5` for day-to-day editing — best correctness-per-dollar and respects the surrounding code style.
- **Hard refactors / multi-file architecture changes:** switch to `claude-opus-4.5` or `gpt-5` (whichever is included in the current Cursor plan tier). Don't waste Opus calls on one-line edits.
- **Auto / cheap mode:** fine for tab-completion and trivial inline edits; never trust it for cross-file changes or anything touching auth, billing, or migrations.
- **Embedding / search:** leave Cursor's default; don't override.
- **Pricing / context limits:** look them up at <https://cursor.com/pricing> and <https://docs.cursor.com/models> rather than guessing. The Claude family pricing also lives in the `claude-api` skill.

## Known bugs / quirks / workarounds

- **`.cursor/rules/*.mdc` requires the YAML frontmatter** (`alwaysApply: true`) to be applied globally. Without it, the file is silently ignored. Per-path rules use `globs:` in the same frontmatter — never both `alwaysApply: true` and `globs:` together.
- **Legacy `.cursorrules`** is still honoured but deprecated; do not create new ones. Migrate any survivors into `.cursor/rules/`.
- **`AGENTS.md` fallback:** Cursor reads a top-level `AGENTS.md` natively when no `.cursor/rules/` exists, so this stub is belt-and-braces.
- **Windows + Defender ASR:** unsigned `postinstall` native binaries from npm get blocked silently (exit 1, no message). If a Cursor extension or MCP server install fails on Windows with no obvious cause, check Event Viewer → Defender → Operational, then either install with `--ignore-scripts` or add a narrow ASR exclusion.
- **Indexing on large repos:** Cursor's codebase index can stall on repos >50k files. Add big generated dirs (`dist/`, `.next/`, `node_modules/`, `coverage/`) to `.cursorignore`.
- **Composer "Apply" sometimes corrupts large files** — review the diff before accepting; if the apply mangles indentation, fall back to manual edits.

## Edit-mode and tool preferences

- **Composer (Agent mode)** for anything multi-file. **Chat mode** for read-only questions and design discussion. **Inline edit (Cmd+K / Ctrl+K)** for surgical single-region edits.
- **Always run with the codebase indexed** before asking architectural questions — un-indexed answers hallucinate file paths.
- **Auto-run / YOLO mode:** OFF by default. Enable per-session only for sandboxed throwaway repos. Never on `chirag127/secrets` or anything with production credentials.
- **Terminal:** Cursor's integrated terminal on Windows must be set to Git Bash (`C:\Program Files\Git\bin\bash.exe`), not PowerShell or cmd. POSIX commands assumed throughout `~/AGENTS.md`.
- **Don't auto-commit.** Cursor's "commit message" feature is fine for drafting, but I review and run `git commit` myself per the conventional-commits rule.

## Skills + MCP servers

Cursor speaks MCP via `~/.cursor/mcp.json` (user-global) or `<repo>/.cursor/mcp.json` (per-project). Keep secrets out of the per-project file — reference env vars instead.

- **Personal skills** are not installed into Cursor directly; they live in [`chirag127/setup`](https://github.com/chirag127/setup) and are invoked from Claude Code. For Cursor, replicate the behaviour as `.cursor/rules/*.mdc` snippets when needed.
- **Recommended MCP servers** (install via the Cursor settings UI → MCP, or hand-edit `~/.cursor/mcp.json`):
  - `@modelcontextprotocol/server-github` — repo / issue / PR access (`GITHUB_PERSONAL_ACCESS_TOKEN` from `chirag127/secrets`).
  - `@modelcontextprotocol/server-filesystem` — scoped to `C:\D` only; never the whole drive.
  - `@upstash/context7-mcp` — up-to-date library docs, free.
  - `@playwright/mcp` — browser automation for E2E debugging.
