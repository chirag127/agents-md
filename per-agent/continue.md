# Continue rules — Chirag Singhal

> **Read [`~/AGENTS.md`](file://~/AGENTS.md) first** — that file holds the shared rules across every coding agent. This file adds Continue-specific rules on top: model defaults, known bugs and workarounds, edit-mode preferences, and the skills / MCP servers installed for this agent.

## Model defaults — which models to use

- **Chat / edit:** route through my own free aggregators rather than paying per-token. Primary is [`chirag127/freellmapi`](https://github.com/chirag127/freellmapi) (OpenAI-compatible proxy over free upstream tiers); fall back to [`chirag127/OmniRoute`](https://github.com/chirag127/OmniRoute) when freellmapi rate-limits.
- **Configure as an OpenAI-compatible provider** in `~/.continue/config.yaml`: set `provider: openai`, `apiBase: http://localhost:<port>/v1` (or the deployed URL), and the model name the proxy advertises (e.g. `gemini-2.5-pro`, `deepseek-chat`, `glm-4.6`).
- **Local models:** Ollama for offline work — `qwen2.5-coder:7b` for autocomplete, `qwen2.5-coder:32b` for chat when the laptop has the headroom. Use `provider: ollama` directly, no proxy needed.
- **Autocomplete model** must be a small fast one (`qwen2.5-coder:1.5b` or a hosted free tier). Never point autocomplete at a slow chat model — Continue will hammer it on every keystroke.
- **Embeddings:** `nomic-embed-text` via Ollama for the local index. Avoid OpenAI embeddings — paid and unnecessary for personal codebases.

## Known bugs / quirks / workarounds

- **Windows 11 + Defender ASR:** the VS Code extension itself installs fine, but any MCP server Continue spawns via `npx` postinstall can be blocked if it ships an unsigned native exe. Symptom: server status stays "starting" forever, no useful log. Workaround: pre-install the MCP server globally with `pnpm add -g <pkg>` from a Git Bash session, then point Continue at the resolved binary.
- **Two rules formats both load:** `.continuerules` (legacy single file at repo root) and `.continue/rules/*.md` (current directory form) are both honoured and **merged**. Don't keep both — pick the directory form for new repos so rules can be split per-concern.
- **`config.yaml` vs `config.json`:** YAML is the current format; the JSON form still parses but new features (rule blocks, agent mode tools) only land in YAML. Migrate when touched.
- **Indexing chokes on monorepos:** add large generated dirs to `.continueignore` (same syntax as `.gitignore`) — `node_modules`, `.next`, `dist`, `coverage`, plus any vendored submodules from `chirag127/setup`.
- **`@codebase` retrieval is lossy** on repos > ~5k files; prefer `@folder` or `@file` with explicit paths for anything load-bearing.

## Edit-mode and tool preferences

- **Agent mode** (the one with tool calls) over Chat mode for any change that touches more than one file. Chat mode for explanations / one-shot edits.
- **Apply edits inline** via the diff view — never accept a wholesale file rewrite without reading the diff first. Continue's "apply" can silently drop import lines on large files.
- **Custom slash commands** live in `config.yaml` under `prompts:` — keep them in the user-level config, not per-repo, unless the prompt is genuinely repo-specific.
- **Match surrounding code style and prefer editing existing files** (per `~/AGENTS.md`). Continue's default prompt is a bit eager to create new helper files — push back when it does.

## Skills + MCP servers

Continue's MCP support is configured under `mcpServers:` in `config.yaml`. Recommended set, free-first:

- **filesystem** — `npx -y @modelcontextprotocol/server-filesystem <root>` for scoped file access.
- **git** — `uvx mcp-server-git --repository <path>` for diff/log/blame queries without shelling out.
- **github** — `npx -y @modelcontextprotocol/server-github`, env `GITHUB_PERSONAL_ACCESS_TOKEN` from [`chirag127/secrets`](https://github.com/chirag127/secrets).
- **fetch** — `uvx mcp-server-fetch` for pulling docs/URLs into context.

Skills are not a first-class Continue concept the way they are in Claude Code; the closest equivalent is the `prompts:` section of `config.yaml`. Mirror the most-used Claude Code skills (e.g. README-first, conventional commits) as named prompts here.
