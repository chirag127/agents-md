# Aider rules — Chirag Singhal

> **Read [`~/AGENTS.md`](file://~/AGENTS.md) first** — that file holds the shared rules across every coding agent. This file adds Aider-specific rules on top: model defaults, known bugs and workarounds, edit-mode preferences, and the skills / MCP servers installed for this agent.

## Model defaults — which models to use

- Primary: route through `chirag127/freellmapi` or `chirag127/OmniRoute` via the OpenAI-compatible endpoint. Set `OPENAI_API_BASE` to the proxy URL and `OPENAI_API_KEY` to the proxy key in `~/.aider.conf.yml` (`openai-api-base:` / `openai-api-key:`) — never bake a paid Anthropic / OpenAI key in directly.
- When the proxy is down: fall back to whatever Copilot / Gemini free-tier endpoint is reachable. Confirm with me before switching to a metered key.
- Architect mode (`--architect`): use the strongest available model (currently `claude-opus-latest` style routing via the proxy) for planning, and a cheaper editor model (`gpt-4o-mini`-equivalent) for the actual diff write — saves spend on long sessions.
- `--weak-model` for commit messages and summaries; never the main model.
- Never pass `--no-verify-ssl` unless I explicitly say so.

## Known bugs / quirks / workarounds

- **Windows Git Bash + ASR:** Defender Exploit Guard ASR sometimes blocks Aider's `playwright` / `tree-sitter` native bits when they ship as unsigned exes via npm/pip postinstall. Symptom: `[WinError 5] Access is denied` on first run. Workaround: install inside an admin shell once, or pin the version that has prebuilt wheels.
- **Whole-file edit format on large files** silently truncates when context window fills. Switch to `--edit-format=udiff` (or `diff`) for any file > ~600 lines.
- **`/add` on globs** under Git Bash needs single quotes (`/add 'src/**/*.ts'`) or the shell expands them before Aider sees them.
- **`.aider.conf.yml` location:** Aider looks at repo root first, then `$HOME`. Per-repo overrides win — don't be surprised when a project file shadows the global one.
- **`aider --upgrade`** occasionally pins a broken `litellm` minor; if a model suddenly 400s after upgrade, downgrade `litellm` first before blaming the proxy.

## Edit-mode and tool preferences

- Default `edit-format: whole` for small TS / Python files; `udiff` for anything large or for refactors that touch many files.
- `auto-commits: false` — I want to review the diff and write the conventional-commits message myself. Aider's auto-messages are too verbose.
- `dirty-commits: false` — never commit unrelated working-tree changes.
- `gitignore: true`, `auto-lint: true` with `lint-cmd: "biome check --write"` for TS and `ruff check --fix"` for Python.
- `test-cmd:` set per-repo, not globally — most repos use `pnpm test` or `uv run pytest`.
- Read-only context files via `read: [~/CONVENTIONS.md]` — that's the canonical landing spot for this file (see below).

## Skills + MCP servers

Aider has no MCP client and no skills system of its own. The augmentation surface is:

- **`~/CONVENTIONS.md`** — this file, loaded via `read:` in `~/.aider.conf.yml`. Generated by `skill-agents-md-sync`.
- **`/run` and `/test`** — Aider's built-in shell hooks; use these instead of bolting on an MCP-style runner.
- **External skills** stay in their home agents (Claude Code, Codex). The full 36-skill global inventory is in `per-agent/claude.md`. When a skill's logic is needed inside Aider, paste the relevant snippet into the chat or run the skill in another terminal and feed the output back via `/paste`.
- **OKF knowledge bundles:** per `~/AGENTS.md`, durable repo knowledge is captured as `knowledge/` OKF v0.1 bundles. When entering a repo in Aider, add `knowledge/*.md` via `/read-only` to give Aider passive access to the context layer without consuming edit tokens.
- Repo-level: `chirag127/freellmapi` and `chirag127/OmniRoute` are the proxy backends — install per their READMEs, not via Aider.
