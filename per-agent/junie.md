# JetBrains Junie rules — Chirag Singhal

> **Read [`~/AGENTS.md`](file://~/AGENTS.md) first** — that file holds the shared rules across every coding agent. This file adds Junie-specific rules on top: model defaults, known bugs and workarounds, edit-mode preferences, and the skills / MCP servers installed for this agent.

## Model defaults — which models to use

- Junie picks its own backend model per-task; there is no user-facing model selector in the IDE plugin as of mid-2026. Treat it as a fixed agent and don't try to override.
- For tasks that need a specific frontier model (Opus-class reasoning, long-context refactors > 200k tokens), bail out of Junie and use Claude Code or Codex CLI instead — they're configured for that in `~/AGENTS.md`. Junie is best for "stay inside the IDE, touch a few files, run the tests" loops, not multi-repo planning.
- AI Assistant is a separate JetBrains product. If a chat panel inside the IDE answers but doesn't act on the project, it's AI Assistant, not Junie — close it and use the Junie tool window.

## Known bugs / quirks / workarounds

- **High autonomy by default.** Junie will edit files, run terminal commands, and re-run tests without asking. Combined with the "confirm before hard-to-reverse" rule in `~/AGENTS.md`, this means: never let Junie run `git push`, `git push --force`, `rm -rf`, deploy commands, or anything that hits cloud storage / external APIs without explicit per-turn authorisation. When in doubt, switch Junie from Code mode to Ask mode.
- **No global config.** Junie reads `<repo>/.junie/guidelines.md` from the project root only. There is no `~/.junie/` equivalent. The `agents-md-sync` skill writes the per-repo file; if it's missing, Junie falls back to no rules at all (silent — no warning).
- **Windows 11 + Defender ASR.** Junie's bundled JCEF runtime is signed and unaffected, but any `npm install` Junie kicks off in the terminal hits the same ASR block as Claude Code: unsigned native postinstall binaries (`esbuild`, `sharp`, `better-sqlite3`) get killed silently. Workaround: pre-install with `pnpm install --ignore-scripts` in Git Bash, then `pnpm rebuild` for the specific package, or whitelist the project dir in Defender.
- **Test-runner detection is brittle on monorepos.** Junie will sometimes run `npm test` at the repo root when the actual package is in `packages/<x>`. Pin the test command in `.junie/guidelines.md` per repo.
- **Long-running terminal commands** (dev server, `playwright test --ui`) wedge the agent — it waits for exit. Use `nohup ... &` or run them in a separate Git Bash window outside Junie.

## Edit-mode and tool preferences

- **Default to Code (Brave) mode** for routine edits; **Ask mode** for any task touching `git`, deploy configs, secrets, or the `chirag127/secrets` repo.
- **One concern per Junie session.** Don't chain "fix bug + add feature + refactor" — Junie's diff gets unreviewable. Stop the agent, commit, start a new task.
- **Always read the file before editing.** Same rule as `~/AGENTS.md`: surrounding-style match beats house style.
- **Forks:** the fork-minimal rule from `~/AGENTS.md` applies — Junie tends to "tidy" upstream files. Tell it explicitly "do not modify any file tracked by upstream" at the top of the task prompt.
- **Conventional commits, small commits.** Junie's auto-commit message defaults are fine; verify the type (`feat:` / `fix:` / `chore:`) matches the change.

## Skills + MCP servers

- **Skills:** Junie does not currently load Anthropic-style skills or `bunx skills` packages. The `chirag127/skill-agents-md-sync` skill runs **outside** Junie and writes `.junie/guidelines.md` into each repo — that's the only integration point.
- **MCP servers:** Junie added MCP client support in the 2025.x line. Configure per-project via `.junie/mcp.json` (same JSON shape as Claude Code's `mcpServers`). Keep the server list short — Junie loads every tool into the same prompt and longer lists slow it down. Default minimum: `filesystem` (scoped to repo root) and `git`. Add `playwright` only for repos that actually run E2E.
- Run `node ~/src/skill-agents-md-sync/scripts/sync-agents.mjs` after editing this file to push changes to every repo's `.junie/guidelines.md`.
